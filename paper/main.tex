\begin{abstract}
本文面向语音信号处理课程大作业，围绕开源神经语音合成模型 VoxCPM/VoxCPM2 设计并实现了一个可控音色语音合成与语音聊天系统。项目重点在本地语音模型的推理与适配：通过自然语言音色描述、参考音频克隆和 LoRA 说话人微调，实现从文本条件到目标语音波形的生成；对话模型仅用于产生待朗读文本和语气条件，不参与语音信号建模。系统实现包括音频数据预处理、VoxCPM 条件输入封装、推理参数控制、参考音频复用、训练 manifest 构建和 LoRA 配置生成。论文从连续语音表征、层次化语义-声学建模、音色控制与实验评价四个方面分析系统。实验部分计划从实时率、可懂度、音色相似度、主观自然度和不同推理步数下的速度质量权衡进行评估。硬件分析表明，4GB 显存笔记本不适合舒适运行 VoxCPM2，需要采用 CPU 推理、低资源 VoxCPM-0.5B 或更高显存设备。

\textbf{关键词：} 语音合成；VoxCPM；声音克隆；音色控制；LoRA 微调
\end{abstract}

\section{引言}
随着神经网络声学模型、扩散模型和大规模语音生成模型的发展，语音合成已经从传统拼接合成、统计参数合成发展到能够进行零样本声音克隆和自然语言风格控制的生成式系统。课程项目关注深度学习与语音信号处理的结合，本文选择“可控音色语音合成”作为核心任务，并将其放入语音聊天场景中展示：系统接收待朗读文本和语气控制条件，再由本地 VoxCPM 模型生成目标音色的语音波形。

语音合成是语音信号处理的典型任务，其本质是学习从离散文本符号序列到连续声学波形的条件映射。该映射高度欠定：同一句文本可以由不同说话人、不同情绪、不同语速自然地读出，因此可控性（音色、情感、节奏的显式调节）和稳定性（避免漏字、重复、跳读）一直是工程落地的核心难点。近年的生成式语音模型借助大规模预训练，在零样本声音克隆和自然语言风格描述上取得突破，但同时带来了模型体量大、显存需求高、推理较慢等部署代价。课程实验既要体现深度学习方法的先进性，也要在有限算力上跑通完整链路，这正是本文工程化工作的出发点。

本项目的目标不是调用在线语音合成服务，而是复现并工程化一个开源本地语音模型。系统保留 VoxCPM 上游模型作为核心声学生成后端，在项目侧实现音色控制参数封装、参考音频管理、数据预处理、LoRA 配置生成、速度基准测量和命令行推理入口。系统在架构上把“对话内容生成”与“语音信号生成”彻底解耦：聊天文本由兼容 OpenAI 接口的对话模型（默认 DeepSeek）产生，本地不加载任何语言模型，只运行语音合成；论文的技术重点始终是本地语音生成链路：如何组织文本、语气、参考音频和微调权重，使 VoxCPM 输出稳定、可控且可比较的语音。本文的主要工作包括：（1）设计对话与语音解耦的可控音色语音聊天系统并完整实现；（2）封装 VoxCPM 的自然语言音色设计、参考音频克隆与 LoRA 适配三种音色控制方式；（3）实现数据预处理与训练 manifest 构建，给出 LoRA 微调配置生成流程；（4）实现跨推理步数的实时率基准脚本，并在 4GB 显存笔记本上给出低资源部署的可行性分析。

\section{国内外研究现状}
早期语音合成常使用拼接合成和统计参数合成，近年来主流方法转向基于深度神经网络的端到端 TTS。Tacotron~2 将文本经注意力机制映射为梅尔谱，再由 WaveNet 声码器还原波形，显著提升了自然度 \cite{shen2018tacotron2}；FastSpeech~2 引入非自回归并行生成和显式时长、音高、能量预测，在加速推理的同时增强了韵律可控性 \cite{ren2021fastspeech2}；VITS 把变分自编码器、规范化流和对抗训练统一进单阶段端到端框架，实现了高保真直接波形生成 \cite{kim2021vits}。这些工作共同确立了“文本到中间声学表征再到波形”的神经合成范式。

近年的大规模语音生成模型进一步引入离散语音 token、扩散模型、流匹配和语言模型骨干，使系统具备零样本声音克隆、多说话人控制和跨语言生成能力。VALL-E 将 TTS 重构为基于神经音频编解码器离散 token 的语言建模任务，仅凭数秒参考音频即可零样本克隆说话人音色与声学环境 \cite{wang2023valle}。与此同时，扩散与流匹配方法 \cite{lipman2023flowmatching} 为连续声学表征的生成提供了高质量采样框架，无分类器引导（CFG）\cite{ho2022cfg} 则在采样时通过条件与无条件预测的线性组合增强文本条件的约束强度，是控制生成稳定性与表现力权衡的常用手段。

VoxCPM 的核心特点是 tokenizer-free，即不依赖外部离散语音分词器，而是在连续语音表征空间中通过层次化语义-声学建模生成语音 \cite{zhou2025voxcpm}。相比依赖离散 token 的方案，连续表征避免了量化损失和码本受限问题，有利于保留细粒度音色与韵律。VoxCPM2 在此基础上扩展到 2B 参数、30 种语言、自然语言音色设计、可控声音克隆和 48kHz 输出，并以 MiniCPM-4 作为语言骨干 \cite{voxcpm2_2026}。其官方文档给出了 LocEnc、TSLM、RALM、LocDiT 的四阶段结构，这为课程项目提供了一个具备研究价值、且开源可微调的本地语音模型基础。

在交互系统中，对话模型只提供待朗读文本和语气描述，可由 DeepSeek 或其他兼容接口生成 \cite{deepseekai2025r1}。该部分属于上层文本条件生成，不改变语音模型的训练与推理机制；工程实现借助 Hugging Face Transformers 生态完成模型加载与推理 \cite{wolf2020transformers}。因此本文后续重点讨论 VoxCPM 的本地推理、音色控制、参考音频条件和 LoRA 适配 \cite{hu2022lora}，而不是对话 API 本身。

\section{方法}
\subsection{系统总体结构}
系统由四个模块组成：文本与语气条件生成、VoxCPM 本地语音合成、音色/参考音频控制、实验与微调工具链。给定待朗读文本 $y$、语气提示词 $p$、基础音色描述 $s$ 和可选参考音频 $a_r$，系统首先构造 VoxCPM 的条件文本：
\[
\tilde{y} = \mathrm{concat}((s, p), y).
\]
其中括号内的自然语言描述用于控制音色、情绪、语速和停顿。随后 VoxCPM 根据文本条件、可选参考音频和可选提示音频转写生成波形：
\[
\hat{a} = g_{\phi}(\tilde{y}, a_r, a_p, t_p).
\]
其中 $a_p$ 和 $t_p$ 分别表示极致克隆模式下的提示音频与对应转写文本。项目侧代码不修改 VoxCPM 主体结构，而是实现条件组织、模型加载、输出保存和实验参数控制。

为了保证对话内容与语音生成解耦，系统约束对话模型只通过 Chat Completions 的工具调用（function call）返回结构化结果，包含三个字段：$y$（\texttt{spoken\_text}，真正朗读的文本）、$p$（\texttt{voice\_prompt}，仅本轮的情绪、语速、停顿与重音描述）和一个不朗读、不进入 TTS 的对话状态摘要 $c$（\texttt{dialogue\_state}）。$c$ 在多轮之间持续传递，用于保持人设、关系进展和表达基线的连续性，但绝不作为语音合成提示词。系统提示词显式禁止输出思维链与内心独白，需要跨轮保留的信息一律写入 $c$。这样既利用了对话模型的角色规划能力，又确保本地语音链路只接收明确的朗读文本与语气条件。本地不加载任何语言模型，符合“本地只跑语音生成”的设计定位。

\subsection{VoxCPM 连续表征语音生成}
VoxCPM 的关键思想是避免依赖外部离散语音分词器，直接在连续音频隐空间中建模语音。其整体流程可概括为四个阶段：局部编码器将音频片段映射到紧凑连续表征；文本-语义语言模型负责根据文本捕捉语义、韵律和停顿计划；残差声学语言模型补充细粒度声学信息；局部扩散 Transformer 在条件流匹配或扩散解码框架下生成连续音频 latent，最后由 AudioVAE 解码为波形。VoxCPM2 进一步支持 48kHz 输出、多语言合成、自然语言音色设计和参考音频可控克隆。

本项目的推理封装集中在三个参数：CFG 引导系数、扩散推理步数和最大生成长度。CFG（无分类器引导）\cite{ho2022cfg} 在采样时按系数线性放大文本条件相对于无条件预测的方向，系数越大文本约束越强、表现力越克制；扩散/流匹配解码 \cite{lipman2023flowmatching} 的推理步数决定了从噪声到目标 latent 的离散积分精度，步数越多通常质量越高但耗时线性增长；最大长度则控制生成音频时长和显存占用。本文实验正是围绕推理步数这一关键超参，量化其对实时率与质量的影响。为了适应课程实验和低显存机器，默认关闭 denoiser 与编译优化，优先保证模型可加载和流程可复现。

\subsection{VoxCPM 语音合成与音色控制}
VoxCPM2 支持两种主要控制方式。第一，自然语言音色设计，即在合成文本前加入括号形式的描述，例如“年轻女性，声音温柔自然，语速中等”。第二，参考音频克隆，即提供参考语音，使模型生成接近参考说话人的音色。项目将用户指定的基础音色或参考音频作为全局设置，在连续对话中保持不变；每轮语气提示词只调整情绪、语速、停顿和重音，不改变说话人身份。

项目在 \texttt{VoxCPMSynthesizer} 中统一封装模型加载、LoRA 权重加载、条件文本拼接和音频文件输出。对单句合成，输入为文本和音色描述；对参考音频克隆，输入额外包含 \texttt{reference\_audio}；对极致克隆，输入包含 \texttt{prompt\_audio} 与 \texttt{prompt\_text}。最终输出为 WAV 文件，采样率由 VoxCPM 模型本身决定。

\subsection{LoRA 微调}
为了适配特定说话人，项目提供 CSV/JSONL 到 VoxCPM manifest 的预处理脚本，并生成上游 LoRA 微调配置。预处理阶段将原始音频统一转换为单声道 16kHz WAV，过滤过短或过长样本，并写出包含音频路径、转写文本、时长和可选说话人编号的 JSONL manifest。LoRA \cite{hu2022lora} 通过向权重注入低秩增量矩阵，仅更新少量适配参数即可完成说话人适配，较适合课程实验中的小规模自录数据。微调完成后，推理阶段通过 \texttt{lora\_path} 加载适配权重，从而在保留基础模型能力的同时增强目标说话人音色。

\section{实验设置与结果分析}
\subsection{实验环境}
当前开发机器为 Intel i7-12650H、13GB RAM、NVIDIA RTX 3050 Laptop GPU 4GB。软件环境为 Python 3.11、uv、PyTorch 2.5+、Transformers 和 VoxCPM 上游代码。值得注意的是，本机自检（\texttt{voiceagent doctor}）显示 \texttt{cuda\_available=false}：所安装的 PyTorch 构建需要 CUDA 12+，而本机 NVIDIA 驱动仅为 CUDA 11.8，版本过旧无法被该 torch 使用。这意味着即便忽略显存限制，当前软件栈也只能走 CPU 推理。

\subsection{低资源部署分析}
依据上游显存需求（VoxCPM2 约 8GB、VoxCPM1.5 约 6GB、VoxCPM-0.5B 约 5GB），4GB 显存不足以装下任何一档模型，再叠加上述驱动-框架不匹配，本机的现实部署路径是 CPU 推理：可以合成短句用于功能演示，但 2B 扩散自回归模型在 CPU 上的实时率（RTF）远大于 1，无法实时聊天。为此项目在 \texttt{doctor} 命令中实现了自动档位推荐：依据检测到的显存与内存，在 VoxCPM2/VoxCPM1.5/VoxCPM-0.5B 与 CPU/GPU 之间给出建议，本机的推荐结果为 “VoxCPM-0.5B + CPU（速度优先）”。这一分析说明：在课程级低资源设备上复现前沿语音模型，模型选型、推理步数与设备选择之间存在明确的工程权衡，而非简单地追求最大模型。

\subsection{数据与任务}
实验计划使用两类数据。第一类为自录普通话语音，时长 5--10 分钟，按句切分并人工转写，用于 LoRA 适配。第二类为开源中文 TTS 数据集中的少量样本，用于对比不同音色控制设置下的合成效果。若使用公开数据集，需在最终提交中补充数据集许可证与下载地址。

\subsection{评价指标与基准方法}
实验将记录以下指标：

\begin{itemize}
  \item 实时率 RTF：合成耗时除以生成语音时长，越低越好。
  \item 字错率/词错率：使用 ASR 转写合成音频并与目标文本比较。
  \item 音色相似度：使用说话人嵌入模型计算参考音频与生成音频相似度，或进行主观 MOS 打分。
  \item 主观自然度：从清晰度、自然度、情感一致性三个维度进行 1--5 分打分。
\end{itemize}

其中 RTF 由项目内置的 \texttt{voiceagent bench} 命令自动测量：模型只加载一次并复用，对一组扩散推理步数（如 4、6、10、20）分别合成同一文本、记录墙钟合成耗时与音频时长，计算 RTF 并输出可直接粘入论文的表格。该脚本将推理步数与速度-质量权衡的考察标准化，避免一次性加载开销污染计时结果。

\section{讨论与局限}
本系统的优势在于对话与语音的清晰解耦：对话模型只产出结构化的朗读文本与语气条件，本地仅运行语音合成，既便于替换不同对话服务，也保证了语音链路的可控与可复现。音色控制方面，自然语言音色设计、参考音频克隆和 LoRA 适配三条路径互补，覆盖了从“凭描述造音”到“克隆特定说话人”的不同需求。局限主要来自算力：在 4GB 显存且驱动-框架不匹配的设备上只能 CPU 推理，2B 模型的 RTF 较高，难以支撑实时交互；LoRA 适配也受限于自录数据规模与质量。此外，本文目前的实验数值尚为占位，主客观指标需在补全权重与样本后回填。后续可考虑：升级 CUDA 驱动或换用与之匹配的 PyTorch 构建以启用 GPU、改用 VoxCPM-0.5B 降低开销、引入流式合成降低首包延迟，以及接入说话人嵌入与 ASR 实现客观评测自动化。

\section{结论}
本文实现了一个基于本地 VoxCPM 的可控音色语音合成与语音聊天系统，覆盖了语音合成推理、音色控制、参考音频克隆、数据预处理和 LoRA 微调配置。系统将对话模型限制为文本和语气条件来源，语音生成模型在本地运行，符合课程对深度学习与语音信号处理结合的要求。硬件分析表明，4GB 显存不足以舒适运行 VoxCPM2，后续实验需要在低资源模型、CPU 推理和高显存设备之间权衡。下一步工作是补充真实音频样例、定量 RTF/ASR 指标、音色相似度、主观评分表，并将生成的科研图替换到论文中。
