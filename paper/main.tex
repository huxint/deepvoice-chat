\begin{abstract}
语音合成（Text-to-Speech, TTS）的目标是将文本序列转换为自然、清晰且具有指定说话人特征的语音波形。随着神经声学模型、神经声码器、扩散模型和大规模语音生成模型的发展，TTS 已从传统拼接与统计参数合成，演进到支持零样本声音克隆、自然语言音色控制和多语言生成的端到端系统。本文面向语音信号处理课程实践，基于开源模型 VoxCPM/VoxCPM2 设计并实现一个可控音色语音合成与语音聊天系统。系统将对话文本生成与本地语音合成解耦：对话模型只输出待朗读文本和本轮语气描述，VoxCPM 负责根据文本、音色描述、参考音频或 LoRA 适配权重生成语音。论文重点介绍语音合成的基本技术路线、VoxCPM 的连续表征生成思想、系统架构、音色控制方法、数据预处理与 LoRA 微调流程，并讨论低资源设备和高显存设备上的不同实验策略。结果表明，该系统能够形成完整的“文本条件组织--本地语音推理--音频输出--实验评测”链路；在低显存设备上应优先完成短句演示和流程验证，而实时交互与 LoRA 适配更适合放到高显存平台实测。

\textbf{关键词：} 语音合成；VoxCPM；声音克隆；音色控制；LoRA 微调
\end{abstract}

\section{引言}
语音合成是语音信号处理和人机交互中的基础任务，其核心问题是学习从离散文本符号到连续语音波形的条件映射。同一句文本可以由不同说话人以不同语速、情绪和韵律自然读出，因此语音合成不仅要求内容可懂，还要求声学质量、自然度、音色一致性和可控性。传统方法通常依赖人工设计的文本分析、韵律预测、声学建模与声码器模块；近年来，深度神经网络和生成式模型显著降低了特征工程负担，使端到端、多说话人和零样本声音克隆成为可能。

本文选择“可控音色语音合成”作为课程项目主题，并将其放入语音聊天场景中验证。系统接收用户输入后，由兼容 OpenAI Chat Completions 接口的对话模型生成待朗读文本和语气提示词，再由本地 VoxCPM 模型合成语音。需要强调的是，本文研究重点不是在线聊天模型，而是本地语音生成链路：如何组织文本、音色描述、参考音频和微调权重，使语音模型输出稳定、可控且便于评价。

本文的主要工作如下：
\begin{enumerate}
  \item 梳理语音合成从传统流水线到神经生成模型的技术路线，并说明 VoxCPM 在连续语音表征建模上的特点。
  \item 设计对话生成与语音生成解耦的系统结构，使本地端只承担语音合成、声音克隆和评测任务。
  \item 封装 VoxCPM 的自然语言音色设计、参考音频克隆和 LoRA 说话人适配三种控制方式。
  \item 实现数据预处理、训练 manifest 构建、LoRA 配置生成和实时率（Real-Time Factor, RTF）基准测量。
  \item 分析低显存平台上的部署限制，给出本地验证与高显存上云实验的分工方案。
\end{enumerate}

\section{国内外研究现状}
\subsection{语音合成技术概述}
从工程流程看，典型 TTS 系统一般包含三类核心环节。第一是文本前端处理，将原始文本规范化，并完成分词、读音、韵律边界和停顿等语言信息预测。第二是声学建模，将文本或音素序列映射为梅尔谱、声学隐变量或离散语音 token 等中间表征。第三是波形生成，即由声码器或解码器生成最终音频波形。传统拼接合成依赖大规模录音库检索和拼接，优点是局部音质高，缺点是灵活性差；统计参数合成使用隐马尔可夫模型等方法预测声学参数，稳定性较好但自然度有限；神经 TTS 则利用深度模型直接学习文本到声学表征、再到波形的映射，成为当前主流方向。

Tacotron~2 将文本通过序列到序列注意力模型转换为梅尔谱，再由 WaveNet 声码器还原波形，显著提升了自然度 \cite{shen2018tacotron2}。FastSpeech~2 采用非自回归结构，并显式预测时长、音高和能量，在提升推理速度的同时增强韵律可控性 \cite{ren2021fastspeech2}。VITS 将变分自编码器、规范化流和对抗训练统一到端到端框架中，能够直接生成高保真语音 \cite{kim2021vits}。这些方法共同推动了 TTS 从多模块流水线走向端到端神经合成。

\subsection{大规模语音生成模型}
近年来，语音生成进一步受到大语言模型、神经音频编解码器、扩散模型和流匹配方法的影响。VALL-E 将语音合成建模为离散音频 token 的语言建模任务，仅凭短参考音频即可进行零样本声音克隆 \cite{wang2023valle}。扩散模型和流匹配方法则通过逐步去噪或连续概率路径学习生成高质量声学表征 \cite{lipman2023flowmatching}；无分类器引导（Classifier-Free Guidance, CFG）可在采样时调节条件约束强度，从而平衡文本一致性与表达自然度 \cite{ho2022cfg}。

VoxCPM 的特点是 tokenizer-free，即不依赖外部离散语音分词器，而是在连续语音表征空间中进行层次化语义--声学建模 \cite{zhou2025voxcpm}。相比离散 token 方案，连续表征能够减少量化误差，保留更细粒度的音色、韵律和情感信息。VoxCPM2 进一步扩展到更大的模型规模、多语言合成、自然语言音色设计、参考音频克隆和 48kHz 输出 \cite{voxcpm2_2026}。这些能力使其适合作为课程项目中的本地语音合成后端。

\subsection{个性化音色适配}
可控音色合成通常有三种实现路径。第一，使用自然语言描述指定音色和风格，例如“年轻女性、语速中等、语气温和”。第二，使用参考音频作为条件，让模型保持参考说话人的音色。第三，在少量目标说话人数据上进行参数高效微调。LoRA 通过在原模型权重旁引入低秩增量矩阵，只训练少量适配参数，能够在保留基础模型能力的同时提高特定说话人音色相似度 \cite{hu2022lora}。本文系统同时支持上述三种方式。

\section{系统设计与实现}
\subsection{总体架构}
系统由四个部分组成：对话文本生成、语音合成推理、音色控制与参考音频管理、实验评测与微调工具链。对话模型只负责产生结构化文本结果，包括真正朗读的 \texttt{spoken\_text}、本轮语气提示词 \texttt{voice\_prompt} 和不进入 TTS 的 \texttt{dialogue\_state}。本地语音端接收朗读文本、基础音色描述、语气提示词、可选参考音频和可选 LoRA 权重，然后调用 VoxCPM 生成 WAV 文件。

给定朗读文本 $y$、本轮语气提示词 $p$、基础音色描述 $s$ 和可选参考音频 $a_r$，系统首先构造语音模型的条件文本：
\[
  \tilde{y}=\mathrm{concat}((s,p),y).
\]
其中 $s$ 用于保持说话人身份和基础音色，$p$ 用于控制本轮情绪、语速、停顿和重音。VoxCPM 的推理过程可抽象为：
\[
  \hat{a}=g_{\phi}(\tilde{y},a_r,a_p,t_p),
\]
其中 $\hat{a}$ 为生成波形，$a_p$ 和 $t_p$ 分别表示极致克隆模式下的提示音频及其转写文本。项目代码不修改 VoxCPM 主体网络，而是在工程侧完成模型加载、条件封装、参数传递、输出保存和评测统计。

\subsection{VoxCPM 语音生成流程}
VoxCPM 的语音生成可概括为“文本语义规划--声学细节补充--连续 latent 生成--波形解码”四个阶段。局部编码器将语音片段表示为紧凑连续特征；文本--语义语言模型根据输入文本预测语义、韵律和停顿；残差声学语言模型补充细粒度声学信息；局部扩散 Transformer 在扩散或流匹配框架下生成连续音频 latent，最后由解码器还原为波形。该流程避免了外部离散 tokenizer 对音频细节的量化限制，有利于保留说话人的音色差异和自然韵律。

在项目实现中，推理质量和速度主要受三个参数影响。CFG 引导系数越大，文本条件约束越强，但语音表现可能更保守；扩散推理步数越多，采样轨迹越精细，通常质量更高但耗时增加；最大生成长度决定输出音频时长和内存占用。项目的基准脚本围绕推理步数测量 RTF，以便直观比较速度和质量之间的权衡。

\subsection{音色控制方法}
系统支持三类音色控制方式：
\begin{enumerate}
  \item \textbf{自然语言音色设计：}用户通过文字描述说话人年龄、性别、音色、情绪和语速，系统将描述拼接到待合成文本前，使模型按描述生成语音。
  \item \textbf{参考音频克隆：}用户提供一段参考语音，模型在合成时模仿参考说话人的音色和声学环境。该方式适合不进行训练的零样本声音克隆。
  \item \textbf{LoRA 说话人适配：}用户准备少量目标说话人音频及转写文本，经预处理生成 manifest 后调用上游训练脚本微调 LoRA 权重，推理时加载该权重以增强个性化音色。
\end{enumerate}

为了保持多轮对话中的声音一致性，系统把基础音色描述或参考音频作为全局条件保存；每轮对话模型只生成语气提示词，不能覆盖说话人身份。这样可以减少不同轮次之间音色漂移的问题。

\subsection{数据预处理与 LoRA 配置}
预处理脚本读取 CSV 或 JSONL 元数据，将音频统一转换为单声道 16kHz WAV，并过滤时长过短或过长的样本。输出 manifest 每行包含音频路径、转写文本、时长和可选说话人编号，格式示例如下：
\begin{lstlisting}
{"audio": "data/processed/wavs_16k/sample001_16000.wav",
 "text": "你好，这是第一条训练语音。",
 "duration": 3.2}
\end{lstlisting}

LoRA 配置生成脚本根据预训练模型路径、训练 manifest、输出目录和训练超参数生成 YAML 配置。实际训练仍调用 VoxCPM 上游脚本完成，项目侧负责把课程实验数据整理为上游代码可直接读取的格式。

\subsection{评价指标}
本文采用以下指标评价系统：
\begin{enumerate}
  \item \textbf{实时率 RTF：}合成耗时与生成音频时长之比，数值越小速度越快；RTF 小于 1 表示合成速度快于实时播放。
  \item \textbf{文本一致性：}使用自动语音识别转写合成音频，并与目标文本计算字错率或词错率。
  \item \textbf{音色相似度：}使用说话人嵌入模型计算参考音频与生成音频的相似度，或由人工进行相似度打分。
  \item \textbf{主观自然度：}从清晰度、自然度、情感一致性三个维度进行 1--5 分主观评分。
\end{enumerate}

项目内置的 \texttt{voiceagent bench} 命令用于测量不同扩散推理步数下的 RTF。基准程序只加载一次模型，依次在 4、6、10、20 等步数下合成同一测试文本，记录墙钟耗时、音频时长和 RTF，避免模型加载时间污染推理速度统计。

\subsection{低资源部署分析}
根据 VoxCPM 上游说明，VoxCPM2、VoxCPM1.5 和 VoxCPM-0.5B 的显存需求大致分别为 8GB、6GB 和 5GB。低显存设备不足以稳定运行这些模型，CPU 模式可以验证完整流程，但 2B 级别语音生成模型的 RTF 通常远大于 1，难以支撑实时聊天。因此，低资源设备上的合理策略是：优先完成命令行、预处理、结构化对话和短句 CPU 演示；真实的速度、音色克隆和微调实验放到高显存 GPU 上完成。

云端实验建议选择至少 16GB 显存的 GPU。若目标是完整展示 VoxCPM2 的自然语言音色设计和 48kHz 输出，可优先使用 VoxCPM2；若目标是尽快获得课程展示结果，可先用 VoxCPM-0.5B 跑通 RTF、参考音频克隆和少量 LoRA 适配，再补充 VoxCPM2 的质量优先样例。推理步数建议设置为 4、6、10、20 四档，观察速度与自然度之间的变化。

\section{讨论}
本系统的主要优势是结构清晰、模块边界明确。对话模型只产生文本和语气条件，本地端只运行语音合成模型，因此对话服务、语音模型和评测工具可以独立替换。音色控制方面，自然语言描述适合快速设计声音，参考音频适合零样本克隆，LoRA 适合固定目标说话人的个性化适配，三者覆盖了从演示到定制的不同需求。

系统的主要局限来自算力和数据。首先，VoxCPM2 参数规模较大，低显存设备难以实时运行；其次，声音克隆效果依赖参考音频质量，背景噪声、混响和转写错误都会影响稳定性；再次，LoRA 微调需要足够干净、覆盖多种语音内容的目标说话人数据，否则可能出现过拟合、发音不稳或情感表达不足。后续可从四个方向改进：迁移到高显存 GPU 完成真实语音实验，接入更小或量化后的模型，增加 ASR 与说话人嵌入自动评测，补充多说话人和多风格样本进行系统比较。

\section{结论}
本文围绕语音合成技术实现了一个基于 VoxCPM 的可控音色语音聊天系统。论文首先概述了 TTS 从文本前端、声学建模到波形生成的基本流程，并介绍了端到端神经 TTS、离散音频 token、扩散/流匹配和连续语音表征等关键技术；随后给出系统架构、VoxCPM 推理封装、音色控制、数据预处理和 LoRA 适配流程。实现结果表明，该系统能够形成完整的语音合成实验链路，并支持自然语言音色设计、参考音频克隆和个性化微调。当前低资源环境适合完成短句演示与流程验证；若要达到实时聊天效果并获得完整定量结果，需要迁移到高显存设备进行补测。
